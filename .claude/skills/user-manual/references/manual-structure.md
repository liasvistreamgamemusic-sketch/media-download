# manual-structure.md — 操作マニュアルの章構成テンプレートと文体規則

## 1. 章構成 (この順序を崩さない)

| # | 章 | スライドパターン | 内容 |
| --- | --- | --- | --- |
| 1 | 表紙 | 表紙 | 製品名・「操作マニュアル」・版数・発行日・対象範囲 (範囲限定時のみ) |
| 2 | 改訂履歴 | 表 | 版・日付・変更内容。初版でも 1 行書く (差分更新の受け皿) |
| 3 | 本書について | 画面概要の変形 | 対象読者 / 動作環境 (対応ブラウザ等・コードから確認できた範囲のみ) / 表記ルール (「」= 画面の文言、① = 画面上の位置) |
| 4 | 画面マップ | 画面マップ | 全画面を章にグルーピングした索引。「やりたいこと → 章」の逆引きにもなる |
| 5〜 | 機能章 × N | 章扉 → 画面概要 → 操作手順 | 下記「機能章の内部構成」 |
| N+1 | こんなときは (FAQ・トラブル) | 表 or 手順 | 「〜できない」「〜が表示される」→ 対処。ログイン不可・入力エラー・権限エラーなど、スクショ/コードで確認できた事象のみ |
| N+2 | 用語集 | 表 | 画面に出る用語のうち一般ユーザーに馴染みがないもの。画面の文言と一字一句同じ表記で載せる |
| N+3 | 問い合わせ先 | 本文 | ユーザーから指定があれば。なければ「(記載してください)」のプレースホルダを置き、報告で伝える |

### 機能章の内部構成

章 = ナビゲーション構造・URL 階層から決めた画面グループ (例: 「注文管理」「設定」)。

1. **章扉**: 章番号 + 章題 + この章でできること 2〜3 行
2. **画面概要** (画面ごとに 1 枚): 画面名 / スクショ / できることの箇条書き / 開き方 1 行
   (例: 「メニューの「注文」をクリックすると表示されます」)
3. **操作手順** (タスクごとに 1 枚〜): タスク名をタイトルにする (「注文を検索する」)。
   コールアウト付きスクショ + 番号手順。**タスク指向で書く** — 画面の全ボタンを説明するのではなく、
   ユーザーがやりたいこと単位で手順化する。使用頻度の高い順に並べる。

## 2. 文体規則

- **敬体 (です・ます)**。操作は「〜をクリックします」「〜を入力します」、
  結果は「〜が表示されます」「〜に戻ります」。
- **1 手順 1 動作**。「A を入力して B をクリックし、C を確認します」→ 3 手順に分ける。
- **操作 → 結果をセットで書く**。結果の書けない手順は根拠不足 — スクショかコードで確認してから書く。
- 画面上の文言は**一字一句そのまま**「」で書く (ボタンが「登録」なら「保存」と書かない)。
- 位置の説明はコールアウト番号で行う。「右上の方にある」のような曖昧表現をしない。
- 条件分岐は「〜の場合:」で明示する。「適宜」「必要に応じて」「基本的に」は禁止。
- 読者を主語にしない受動態を避ける (「クリックされると」→「クリックすると」)。

### 専門用語ブラックリスト (一般エンドユーザー向けのとき)

本文で使ったら言い換える。読者が「社内オペレーター」「管理者」なら業務用語・管理用語は許容:

| 禁止 | 言い換え |
| --- | --- |
| API / エンドポイント / リクエスト | (挙動の説明自体を消す — 操作と結果だけ書く) |
| コンポーネント / モーダル / ダイアログ | 画面 / 小さい画面 / 確認画面 |
| バリデーション (エラー) | 入力内容のチェック (エラー) |
| セッション / トークン | ログイン状態 |
| デフォルト | 最初の設定 / はじめから入っている値 |
| ステータス | 状態 (ただし画面の表記が「ステータス」ならそれに従う) |
| リロード / キャッシュ | ページの再読み込み / (説明を工夫する) |

原則: **画面に表示されている言葉はそのまま使ってよい** (画面が「ステータス」と言うなら本文も「ステータス」)。
画面に出てこない実装用語だけがブラックリスト対象。

## 3. Markdown 版も出す場合

- 章構成・文体は PPTX と完全に同一。画像は相対パスで参照する
  (`docs/manual/images/` にスクショを複製し、`![画面名](images/xxx.png)` で埋め込む)。
- コールアウトは Markdown では再現しないので、手順文側に「画面上部の「登録」ボタン」のように
  位置を言葉で補う (このときも曖昧表現禁止 — 「画面上部の」「一覧の右端の」等、特定できる言い方)。
