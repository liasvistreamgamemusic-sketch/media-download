# git-branch — ボタンで完結するブランチ作成

## 手順

### 1. 状態把握 (質問の前に必ず)

```bash
git status                      # 未コミット変更の有無 (あれば持ち越しになる旨を質問の説明に添える)
git branch --show-current       # 現在ブランチ
git fetch origin                # 起点を最新化
```

- これから何を作るかが会話の文脈で分かっていれば、それに沿った**ブランチ名案**を作っておく

### 2. AskUserQuestion で 1 回だけ質問

| # | 質問 | 選択肢 (先頭=推奨) | 備考 |
| --- | --- | --- | --- |
| 1 | 種別 | 文脈から推測した type を先頭に「(推奨)」+ feat / fix / docs / chore から代替 | プレフィックスになる |
| 2 | 名前 | 「提案: `<種別>/<自動命名案>` (推奨)」 / 「自分で決める」 | 自分で決める場合は「その他」に短い英語ケバブケースで入力してもらう |
| 3 | 起点 | 「origin/main (推奨)」 / 「現在のブランチ (<名前>)」 | デフォルトブランチが main 以外ならそれを先頭に |

### 3. 実行

```bash
git switch -c <type>/<name> origin/main    # 起点が現在ブランチなら origin/main を省く
```

### 4. 報告

- 例: 「`feat/user-stats-api` を origin/main から作成して切り替えました」
- 未コミット変更を持ち越した場合はその旨を 1 行添える
