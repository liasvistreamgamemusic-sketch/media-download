選定後は該当の `<brand>.md` を**全文**読むこと (この索引は選ぶためだけの要約)。

出典: [VoltAgent/awesome-design-md](https://github.com/VoltAgent/awesome-design-md) (MIT)。
各ファイルは実サイトから抽出された実測値 (hex・タイポスケール・余白・角丸・コンポーネント仕様・
Do/Don't・レスポンシブ挙動)。

## 選定の注意

- accent 列はマーケティング面で実質的に差し色として機能する色 (一部ブランドは `colors.primary`
  がインク色のため、本文記載のアクセントを採用している)
- dual は明示的に light/dark 両 canvas を持つブランド。それ以外はネイティブモードのみ —
  逆モードが必要なら Phase 2 で導出合意を取る
- runwayml のみ frontmatter が変則 (Markdown 記述式)。読了時に本文からトークンを起こすこと
- 実プロダクト UI (管理画面・アプリ) を作るときも、マーケティング面の設計言語 (色・タイポ・角丸・
  深度) はそのまま流用できる — コンポーネント密度だけ製品向けに詰める
