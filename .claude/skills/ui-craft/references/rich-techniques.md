# リッチ表現レシピ — 「シンプルなのに豪華」を実装で作る

モーション量「標準〜リッチ」の合意があるときに読む。
前提: リッチさは装飾の物量ではなく、**素材感・タイポの迫力・1 箇所の主役演出**で出す。
どのレシピも anti-ai-checklist.md と motion.md の制約内で使うこと。

## 1. 大型タイポを主役にする (Apple 流・コスト最小)

- ファーストビューは「巨大な見出し + 短い 1 文 + CTA 1 つ」まで削る。
  カード・バッジ・装飾を足さない。サイズ差 (34→56→クランプで 80px 級) が豪華さを作る
- `font-size: clamp(2.5rem, 6vw, 5rem)` でビューポート連動
- 見出しの一部だけ `--color-accent` で着色 (グラデーションテキストにしない)
- スクロールで大見出しが縮んで固定ヘッダーに納まる演出 (Apple の Large Title 挙動) は
  `position: sticky` + scroll-driven animation で再現できる — 主役演出として 1 箇所まで

## 2. 素材感 (フラット単色からの脱却)

- **紙のような背景**: 単色の代わりに、ごく淡いノイズ (SVG feTurbulence を data URI 化,
  opacity 2–4%) や 1% 程度の明度グラデーションを敷く。「なんとなく高い」質感が出る
- **図版の自作**: 画像素材がないとき、灰色ボックスではなく CSS で作る —
  `radial-gradient` 2–3 個の重ねによる柔らかい色面 (紫系は避ける)、
  `repeating-linear-gradient` の細罫パターン、`clip-path` の幾何形状など。
  アクセント色の同系統でまとめると統一感が出る
- **罫線の効かせ方**: エディトリアル風は `1px solid var(--color-border)` の水平罫だけで
  セクションを区切ると、カード無しで高級感が出る

## 3. ガラス・ブラー (使い所を固定する)

意味があるのは「コンテンツの上に浮くレイヤー」だけ:

```css
.floating-header {
  position: sticky; top: 0;
  background: color-mix(in oklch, var(--color-bg) 72%, transparent);
  backdrop-filter: blur(20px) saturate(1.4);
  border-bottom: 1px solid var(--color-border);
}
```

固定ヘッダー・画像上のキャプションバー・オーバーレイメニューまで。**静的なカードには使わない**。

## 4. ホバー・マイクロインタラクション (触感を作る)

- ボタン: hover で `translateY(-1px)` + 影をワンランク上げる、press で `scale(0.98)` —
  「持ち上がって押し込める」物理的な一貫性を作る
- リンク: 下線を `text-decoration-thickness` / `text-underline-offset` のトランジション、
  または背景ハイライトのスライドイン。色がフワッと変わるだけより意図が見える
- 行リスト (テーブル・メニュー): hover で背景ではなく**先頭に 16px 程度のインデント**
  (`padding-left` ではなく内側要素の transform で) + 矢印の出現
- カーソル追従・磁気ボタンは遊び心が要件のときだけ、1 コンポーネント限定

## 5. スクロール演出 (リッチ指定時のみ・各 1 箇所)

- **出現の基本**: セクション単位で fade + rise (8–20px)。子要素 stagger 40ms。
  IntersectionObserver (`threshold: 0.2`, 一度きり) で `.reveal` を付与
- **sticky シーケンス**: 左に固定した図版、右にスクロールするテキストステップ
  (`position: sticky` + Observer でステップ連動の図版切替)。プロダクト説明の主役に 1 回
- **視差**: 背景図版を `animation-timeline: view()` で ±5% だけ動かす。それ以上は酔う
- 文字単位のバラバラ出現・回転入場はやらない (2015 年の LP になる)

## 6. ダーク面の使い方 (全面ダークにしない代替)

ライト基調の中に**ダークのフルブリードセクションを 1 つ**挟むと、緩急でリッチに見える。
その中では色を逆転したトークン (`--color-bg` 等を section スコープで再定義) を使う —
ハードコードで色を差し替えない。

## 7. 数字・データの見せ方

- 統計は「大きい数字 ×3 のバナー」ではなく、文中に組み込むか、1 つの主役数字 +
  補助文で見せる
- カウントアップは主役数字 1 つだけ。`IntersectionObserver` + `requestAnimationFrame`、
  600ms・ease-out、`prefers-reduced-motion` では即値表示
- チャートが要るときは色だけに意味を持たせない (形状・ラベル併用)

## 実装の上限

リッチ指定でも、1 ページに入れる演出は:
主役演出 (sticky シーケンス or 大型タイポ変形) **1** + セクション出現 + マイクロインタラクション。
これを超えて足したくなったら、それは設計ではなく不安。削る。
