# デザイン原則 — Apple HIG 由来の「シンプルなのにリッチ」

Apple HIG の 3 本柱を Web に翻訳したもの。実装前に読み、迷ったらここに戻る。

## 3 本柱

1. **Clarity (明瞭)** — どのサイズでも読める文字、正確なアイコン、装飾は最小限。
   迷ったら要素を足すのではなく削る。
2. **Deference (謙譲)** — UI は後ろに下がり、**コンテンツ (画像・プロダクト・テキスト) が主役**。
   コンテナやボーダーで飾るのではなく、コンテンツ自体を大きく美しく見せる。
3. **Depth (深度)** — 階層はレイヤーと現実的なモーションで伝える。
   影は「触れそう」なくらい柔らかく控えめに (Apple/Stripe 流)。

## タイポグラフィ

**スケールは iOS Dynamic Type を Web に流用する** (実績のある比率):

| 役割 | サイズ (px 目安) | ウェイト |
| --- | --- | --- |
| Display / Large Title | 34–56 (Web では大きめに) | 600–700 |
| Title 1 / 2 / 3 | 28 / 22 / 20 | 600 |
| Headline | 17 | 600 |
| Body (**可読性の床 = 17px**、最低 16px) | 17 | 400 |
| Callout / Subhead | 16 / 15 | 400 |
| Footnote / Caption | 13 / 12 | 400 |

規則:

- 隣接ステップのサイズ比は **1.25 以上** — 差が小さい「フラットな階層」は AI っぽさの一因
- 大型見出しは letter-spacing を詰める (-0.02em〜-0.04em)、本文は 0。
  本文に 0.05em 超のワイドトラッキングをかけない
- 行長 **65–75ch**、line-height 本文 1.5–1.7 / 見出し 1.1–1.2
- 見出し `text-wrap: balance`、本文 `text-wrap: pretty`
- **フォント選定**: 見出し=個性、本文=可読。Inter / Roboto / Arial /
  Space Grotesk / Instrument Serif は「使い古された AI 定番」— デフォルトにしない
  (プロジェクト既定なら従う)。候補: Bricolage Grotesque, Cabinet Grotesk, Clash Display,
  Playfair Display (エディトリアル), 日本語は Noto Sans JP のウェイト対比か
  Zen Kaku Gothic / しっぽり明朝等をトーンで選ぶ

## 色

- **地は静かに、アクセントは効かせる** — ベースはほぼ無彩色 (hue バイアス付き)、
  アクセントは CTA・リンク・主役数字で必ず主張させる (60-30-10)。「控えめ」と「不在」を混同しない
- トークンは**役割名** (semantic)。値名 (`--blue-500`) にしない
- パレットの実値・OKLCH ランプ・コントラスト基準・デュアルテーマ・検証手順は **color-system.md が正** —
  色は目視で選ばず、検証済みプリセットから選ぶか同基準で組んで `validate-palette.mjs` を通す

## 余白・レイアウト

- **4px または 8px グリッド**に全 spacing を乗せる
- 余白は「余り」ではなく**視線を導く設計要素** — セクション間は大胆に (96–160px)、
  関連要素間は詰める。**密→疎のリズム**を作る (全部同じ gap は単調 = AI っぽい)
- 1 画面の主役は 1 つ。重要度でサイズを差別化する — 全要素等価のグリッドにしない
- **1 つの強いレイアウトプリミティブを反復**してサイトの署名にする —
  7 種類のカード装飾を並べない
- タップターゲット **44×44px 以上**、隣接ターゲット間 8px 以上
- モバイルファースト。横スクロールを発生させない。`100dvh` を使う (`100vh` ではなく)

## 深度・素材

- 影は ambient + key の **2 層合成トークン** (実値は rich-techniques.md §2)。
  同一要素に「ヘアラインボーダー + 大きい拡散影」を両掛けしない (どちらか)
- 階層はネストしたカードではなく、**余白・タイポグラフィ・罫線 1 本**で作る
- ガラス表現 (blur) は「上に浮くレイヤー (固定ヘッダー・オーバーレイ)」にだけ意味がある。
  装飾目的で乱用しない (実装レシピは liquid-glass.md)
- 角丸はスケールから選ぶ (rich-techniques.md §3 の 8–28px 言語 + 同心円規則)。
  小さい要素に 24px+ の過剰な丸みをつけない

## インタラクション状態 (全対話要素に必ず定義)

- **hover**: 背景を 5–10% 変える or 前景をわずかに持ち上げる (`translateY(-1px)` 程度)
- **active/press**: `scale(0.97–0.98)` — 周囲のレイアウトをずらさない
- **focus-visible**: アクセント色 2px アウトライン + 2px オフセット。消さない
- **disabled**: `opacity: 0.5` + `cursor: not-allowed`。アニメーションしない

## コンテンツ

- ダミーは「そのドメインの実物らしい」内容にする。金額・日付・人名・商品名を現実的に。
  「Feature 1」「Lorem ipsum」は禁止
- アイコンは **SVG (単一ファミリ・同一ストローク幅・同一サイズ)**。絵文字をアイコンに使わない
- コピーは短く具体的に。「効率化を実現」「世界クラスの」等の空虚な定型句を書かない
